# Migrating to akm v1

## OpenViking source removed

akm v1 removes the `openviking` source provider. The provider required a live
API server for every search and show operation, which prevented the unified
FTS5 indexing pipeline that v1 builds on.

**API-backed sources will return** in a future `QuerySource` tier, designed as
a separate query layer that competes fairly with local FTS5 results. Until that
tier ships, remote API sources are not supported.

### Error you will see

If your config contains an `openviking`-typed source, akm will exit with:

```
openviking is not supported in akm v1. API-backed sources will return as a
separate QuerySource tier post-v1. Remove the source from your config (akm
config sources remove <name>) or downgrade to 0.6.x. See docs/migration/v1.md.
```

### How to remove an OpenViking source

Run the following command, replacing `<name>` with the name shown in
`akm config list`:

```sh
akm config sources remove <name>
```

Then re-run your command. If you have multiple OpenViking sources, repeat for
each one.

To list all configured sources and find the names to remove:

```sh
akm config list
```

### Downgrade path

If you rely on OpenViking integration and cannot wait for the `QuerySource`
tier, downgrade to akm 0.6.x:

```sh
npm install -g akm@0.6
```

Your existing config and stash index are compatible with 0.6.x.

### Timeline

The `QuerySource` tier is planned for a post-v1.0 release. Follow the
changelog at the repository root for updates.
