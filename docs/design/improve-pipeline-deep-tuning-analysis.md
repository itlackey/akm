## Synthesis

# AKM Improve Pipeline — Synthesis Report

## 1. Executive Summary

The pipeline is healthy in throughput but is being measured and tuned against the wrong signals. The highest-leverage work is almost entirely **subtraction and wiring of existing code** — not new machinery. Eight findings dominate:

1. **The success metric is structurally polluted.** `classifyImproveAction` (improve-types.ts:138-149) buckets gated *skips* (distill-skipped, reflect-skipped, reflect-cooldown) as "rejected" — `rejectedActions == distillSkippedActions == 13207` every run. Every accept-rate cited in this whole analysis is downstream of this corruption. Fixing the shared classifier (one site) corrects both `computeImproveRunMetrics` and the `improve_completed` event at once. **This is the prerequisite for trusting any other measurement.**

2. **High accept rates are a safety gate, not a value gate.** consolidate 99.3%, extract 99.9%, reflect 98% are dominated by `--auto-accept` passing a SAFETY check, plus destructive merge/delete ops that never appear in accept-rate at all (consolidate). The real content-accept signal is far lower (distill ~50% once infra rejects are excluded; recombine substance-reject 1/38). Decisions must stop treating "95% accept = value."

3. **Session-checkpoint / telemetry noise (~20% of pool) leaks everywhere except recombine.** The predicates already exist (`isSessionCaptureMemoryName`, `isJunkEntity`) but are applied in exactly one consumer each. Consolidate's pool, the salience high-salience lane, and the entity graph all ingest telemetry that the rest of the system then re-filters or discards. One predicate, reused at 3-4 sites, removes the same noise class system-wide.

4. **The biggest cost waste is LLM calls on guaranteed no-ops.** Extract reaches the LLM on 4,090 sessions to produce 391 candidates (90% `no_candidates`). Recombine spends ~54% of calls on cooldown-discarded output, including re-inducing the 14 already-promoted (terminal) clusters. Distill spends generate+review calls on refs that are already dup-flagged or point at deleted assets. Every one of these has a pre-LLM gate that already exists or is a trivial pre-filter.

5. **The salience scorer cannot tell telemetry from real lessons** because its inputs are inert: `revisionCount` is hardcoded to 0 (distill.ts:958 → `predictionError` pinned at constant 1.0), the novelty vocabulary is built from ref *names* not bodies, and content-salience is scored for `memory:` only (17 of ~4,340 rows). Telemetry checkpoints score 0.84-0.95 alongside genuine lessons. The proactive lane — which sustains all output — doesn't even read the stored content salience (proactive-maintenance.ts:155), ranking a 96%-retrieval-silent pool by type-weight × file-size.

6. **Small-LLM output defects are the top rejection class across three lanes.** Malformed frontmatter, missing/truncated descriptions, and `finishReason='length'` truncations dominate reflect (16+4), consolidate (240/504), and distill rejects — and they fail the *same validators* that already exist in-scope, just post-queue instead of pre-queue. Catching them inside the refine loop is pure waste removal.

7. **Procedural is an empty-pool lane carrying a build-out backlog.** All 6 source entries are feature-docs, not episodes; 5/5 outputs were rejected as over-fit. It is re-enabled in `synthesize.json:12`. The correct move is keep-off + decide remove-or-mothball — not build a miner against zero data.

8. **Several "safety" mechanisms ship default-off or unwired.** Most are dark code that should be enabled (dedup pre-pass, judgedCache, antiCollapse, extract triage). **Correction (maintainer, 2026-06-29):** the proactive-verdict kill-criterion (`scripts/akm-eval/src/proactive-verdict.ts`) is NOT in that category — it is a **deliberate local dev/eval instrument** used to validate the proactive lane's value *before* 0.9.0, and is intentionally **excluded from the release** (do not add it to `src/assets/tasks/`). The only real defect was the doc claim that it "ships enabled"; that has been **retracted** in `improve-optimal-default-config.md`. Proactive ships on based on local verdict validation + production evidence, not a shipped auto-disable.

## 3. Cross-Cutting Themes

**A. Measurement is corrupted at the source.** Three independent forms: (1) gated skips counted as rejections (classifyImproveAction); (2) auto-accept safety pass conflated with value; (3) destructive merge/delete ops invisible to accept-rate (consolidate). Every lane's headline number is suspect until Roadmap #1-2 and the destructive-op counters land. **Fix metrics before tuning anything against them.**

**B. One noise class, many unfiltered consumers.** Session-checkpoint/telemetry (~20% of pool) is excluded only by recombine (#632). Consolidate, the salience high-salience lane, and the entity graph each re-derive or discard it. The fix is always the *same predicate reused at the next consumer* — the canonical net-subtractive pattern.

**C. Wasted LLM spend is concentrated on provable no-ops, and the gates already exist.** Extract no_candidates (90%), recombine cooldown/promoted re-induction (~54%), distill dup/dead-ref rejects, consolidate frontmatter defects. The recurring fix is **move the existing deterministic check before the LLM call**, never "add a new judge."

**D. Salience inputs are inert, so the scorer can't discriminate.** revisionCount:0, ref-name vocab, memory-only scoring, type-stub fallback in the sustaining lane. Wiring real inputs (Roadmap #10-11) must precede any relative-gate redesign, or the gate over-fits the 17 content-scored rows.

**E. Small-LLM output reliability is a top-3 reject cause everywhere.** Malformed frontmatter, missing/truncated descriptions, length-truncation. The validators that reject them already exist and run too late. Pre-queue validation + truncation guards convert silent corruption into clean retries across reflect/distill/consolidate.

**F. Existing machinery ships dark.** dedup pre-pass, judgedCache, antiCollapse, extract triage — all default-off or unwired; enabling what exists beats building new. _(Exception: the proactive-verdict kill-switch is intentionally local-only dev/eval tooling, not release machinery — see Executive Summary #8.)_

**G. Subtract-don't-accrete discipline held under verification.** Multiple originally-"high" recs were correctly downgraded because they added machinery onto unmeasured premises (info-gain LLM gate, procedural miner, per-cycle `maxLlmCalls` budget, embedding coherence gate, percentile salience gate). The pattern of the failure: an inference about net-negative value, "fixed" by a new per-proposal model call.
